**Algorithm** — A step-by-step procedure for solving a problem or making predictions in machine learning. — *Example: We chose a new algorithm to improve the recommendation engine.*
**Model** — A mathematical representation learned from data that makes predictions or classifications. — *Example: The model achieved higher accuracy after we cleaned the dataset.*

**Unsupervised learning** — A method where models discover patterns from unlabeled data. — *Example: We applied unsupervised learning for customer segmentation.*
**Neural network** — A system of connected layers that processes data like a simplified brain. — *Example: The neural network improved our image recognition task.*
**Deep learning** — A type of machine learning using many neural network layers for complex tasks. — *Example: Deep learning helped us recognize speech in the app.*
**Classification** — Predicting a category or label for an input example. — *Example: We used classification to sort incoming support tickets.*

**Clustering** — Grouping similar examples together without predefined categories. — *Example: Clustering revealed new customer segments in the dataset.*

**Underfitting** — When a model is too simple to capture the underlying patterns in data. — *Example: The initial model underfitting showed we needed more features.*
**Validation set** — A separate dataset used to evaluate model performance during training. — *Example: We used the validation set to choose the best model version.*
**Cross-validation** — A technique that splits data into parts to test model stability. — *Example: Cross-validation confirmed the model was reliable across different samples.*
**Natural language processing** — Technology for understanding and generating human language. — *Example: We used natural language processing to analyze customer feedback.*
**Computer vision** — Technology for interpreting visual data such as images and video. — *Example: Computer vision helped us detect defects in the production line.*
**Inference** — Running a trained model on new data to make predictions. — *Example: The inference service returns results in under 200 milliseconds.*
**Bias** — Systematic error in a model caused by unbalanced data or assumptions. — *Example: We audited the training set to reduce bias in the hiring tool.*
**Embeddings** — Numeric representations of words or items that capture semantic similarity. — *Example: We used embeddings to find related product descriptions.*
**Ensemble** — Combining multiple models to improve overall prediction quality. — *Example: The ensemble performed better than any single model in our benchmark.*
**Dropout** — A regularization technique that randomly ignores neural units during training. — *Example: We used dropout to make the model more robust.*
**Activation function** — A function that introduces nonlinearity into neural network predictions. — *Example: We selected ReLU as the activation function for faster training.*
**Loss function** — A formula measuring the difference between predictions and actual values. — *Example: We minimized the loss function to improve the model fit.*
**Gradient descent** — An optimization algorithm that adjusts model parameters to reduce loss. — *Example: Gradient descent helped the model converge during training.*
**Supervised model** — A machine learning system trained with labeled output examples. — *Example: We deployed the supervised model to classify support emails.*
**Transfer learning** — Reusing a pre-trained model for a new but related task. — *Example: Transfer learning allowed us to build a solution faster.*
