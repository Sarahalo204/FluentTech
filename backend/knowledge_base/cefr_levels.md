## A1
1. CEFR "Can Do" descriptor: "I can understand and use familiar everyday expressions and very basic phrases aimed at the satisfaction of needs of a concrete type." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can follow simple greetings, identify basic tools and devices, and respond with brief, rehearsed phrases about their role or tasks. They can understand very short instructions when the language is predictable and supported by visuals.
3. Example sentences:
   1. I am a developer.
   2. This is my laptop.
   3. I need help with this file.
   4. The code is in the project folder.
   5. I use email to talk with the team.
4. Vocabulary:
   - laptop
   - file
   - code
   - app
   - team
   - help
   - project
   - email
   - phone
   - task
5. Typical mistakes:
   - omitting articles (e.g. "I need help with project" instead of "the project")
   - using incorrect word order in simple statements
   - confusing present and past forms
   - choosing overly general words instead of simple tech terms
   - relying on literal translations from their native language

## A2
1. CEFR "Can Do" descriptor: "I can understand sentences and frequently used expressions related to areas of most immediate relevance (e.g. very basic personal and family information, shopping, local geography, employment)." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can understand and describe their position, the software they use, and basic issues they encounter. They can exchange short messages about schedules, meetings, and common operational needs.
3. Example sentences:
   1. I work on the mobile app team.
   2. I have a meeting with the product manager.
   3. My computer is slow today.
   4. I can send the document after lunch.
   5. The server is down, and I need support.
4. Vocabulary:
   - server
   - meeting
   - update
   - install
   - design
   - support
   - mobile
   - schedule
   - document
   - issue
5. Typical mistakes:
   - mixing up simple past and present continuous in status updates
   - using wrong prepositions for time and place
   - confusing similar terms like "install" and "update"
   - forming questions without auxiliary verbs
   - limited use of connectors such as "because" and "so"

## B1
1. CEFR "Can Do" descriptor: "I can understand the main points of clear standard input on familiar matters regularly encountered in work, school, leisure, etc." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can follow routine technical explanations, read basic documentation, and describe problems or progress in meetings. They can participate in everyday team communication and write short emails about project status.
3. Example sentences:
   1. I can explain the bug and how it affects the user experience.
   2. The API response time is slower than expected.
   3. I recommend we test the feature before release.
   4. The QA team found an error in the login flow.
   5. I will update the sprint board after todays stand-up.
4. Vocabulary:
   - bug
   - deploy
   - feature
   - documentation
   - test
   - release
   - feedback
   - deadline
   - sprint
   - user
5. Typical mistakes:
   - using vague vocabulary instead of precise technical terms
   - creating run-on sentences in reports
   - incorrect use of conditionals for planning
   - confusing countable and uncountable nouns
   - inconsistent tense usage in progress descriptions

## B2
1. CEFR "Can Do" descriptor: "I can understand the main ideas of complex text on both concrete and abstract topics, including technical discussions in my field of specialization." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can engage with moderately complex design discussions, understand detailed documentation, and express opinions about architecture or workflow. They can contribute to meetings, ask clarifying questions, and explain solutions clearly.
3. Example sentences:
   1. The architecture diagram shows how the microservices communicate.
   2. We need to refactor the module to improve scalability.
   3. I agree that the deadline should be extended for quality assurance.
   4. The integration tests caught an edge case in the data pipeline.
   5. Please share the user research findings before the next planning session.
4. Vocabulary:
   - architecture
   - scalability
   - refactor
   - integration
   - pipeline
   - deployment
   - stakeholder
   - optimization
   - feedback
   - metrics
5. Typical mistakes:
   - overusing simple connectors instead of more nuanced linkers
   - minor errors in complex sentence structure
   - occasional confusion between similar technical terms
   - lack of precision in expressing probability or risk
   - inconsistent use of passive vs. active voice

## C1
1. CEFR "Can Do" descriptor: "I can understand a wide range of demanding, longer texts, and recognize implicit meaning." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can analyze detailed reports, infer unspoken priorities, and contribute to strategic discussions about product direction. They can summarize complex issues, negotiate trade-offs, and persuade stakeholders with clear argumentation.
3. Example sentences:
   1. Although the prototype performs well, we should consider long-term maintainability and technical debt.
   2. The user analytics indicate that the onboarding flow requires a more intuitive design.
   3. We can balance speed and robustness by introducing feature toggles.
   4. The architecture review highlighted several security vulnerabilities in the authentication service.
   5. I propose a phased rollout to minimize disruption and gather early feedback.
4. Vocabulary:
   - maintainability
   - technical debt
   - prototype
   - analytics
   - intuitive
   - feature toggle
   - vulnerability
   - stakeholder
   - rollout
   - architecture
5. Typical mistakes:
   - occasional subtle errors in advanced grammar use
   - using overly formal or informal tone inconsistently
   - minor problems in expressing nuanced opinions
   - difficulty maintaining perfect coherence in very long explanations
   - slight misuse of idiomatic expressions

## C2
1. CEFR "Can Do" descriptor: "I can understand with ease virtually everything heard or read." (Council of Europe, 2020)
2. In a tech workplace, this means the learner can effortlessly follow high-level strategy discussions, dense technical papers, and fast-paced presentations. They can produce polished proposals, mentor others, and manage communication across global teams without language limitations.
3. Example sentences:
   1. I have reviewed the architectural white paper and recommend a hybrid cloud strategy for resilience.
   2. Our performance benchmarks demonstrate that the new algorithm reduces latency by thirty percent.
   3. The governance framework should align security, compliance, and development priorities.
   4. Having coordinated with the offshore team, I can confirm the migration plan is feasible within the quarter.
   5. The cross-functional review revealed an opportunity to innovate around the customer journey.
4. Vocabulary:
   - resilience
   - benchmark
   - governance
   - compliance
   - migration
   - algorithm
   - optimization
   - cross-functional
   - strategy
   - innovation
5. Typical mistakes:
   - rare slips in register when switching between formal and casual contexts
   - occasional over-elaboration in very long explanations
   - minor idiomatic inaccuracies in highly colloquial speech
   - slight redundancy in written communication
   - negligible issues with extremely specialized jargon
